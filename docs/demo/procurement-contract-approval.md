# 采购合同审批规范

## 目的

为了降低采购合同风险，所有涉及外部供应商的采购合同必须经过系统审批。本文说明采购合同审批所需材料、审批路径、金额节点和常见驳回原因。

## 审批前准备材料

提交采购合同前，业务方需要准备以下材料：

- 采购申请单。
- 供应商报价单或比价记录。
- 合同正文。
- 供应商准入记录。
- 信息安全评估结果，适用于 SaaS、云服务、数据处理类采购。
- 隐私影响评估结果，适用于涉及个人信息处理的采购。

## 审批路径

| 合同金额 | 审批节点 | 预计耗时 |
| -------- | -------- | -------- |
| 小于 10 万元 | 直属负责人、采购 BP | 1-2 个工作日 |
| 10 万元至 100 万元 | 直属负责人、采购 BP、法务 | 2-4 个工作日 |
| 大于 100 万元 | 直属负责人、采购 BP、法务、财务负责人 | 4-7 个工作日 |

## 高风险条款

以下条款不得由业务方自行修改，必须经过法务确认：

1. 违约责任上限。
2. 数据跨境传输。
3. 个人信息处理目的和范围。
4. 供应商审计权。
5. 合同自动续约。

## 常见驳回原因

- 合同主体与供应商准入记录不一致。
- 合同金额与采购申请单金额不一致。
- 未上传比价记录。
- SaaS 服务未完成信息安全评估。
- 涉及用户数据但未填写隐私影响评估。

## 加急审批

采购合同原则上不建议加急。确需加急时，业务负责人需要说明业务影响、上线时间和风险承担人。加急只会缩短审批排队时间，不会跳过法务、财务或安全评估节点。

## Agent 回答提示

当用户询问采购合同、合同审批、供应商准入、SaaS 采购时，需要同时提示是否涉及信息安全和隐私评估。若金额超过 100 万元，需要说明审批节点包括直属负责人、采购 BP、法务和财务负责人。
