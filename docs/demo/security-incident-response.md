# 信息安全事件上报与处置指引

## 适用范围

本文适用于员工发现钓鱼邮件、账号异常登录、敏感数据误发、设备丢失、恶意软件感染等信息安全事件时的上报和处置。

## 事件分级

| 级别 | 场景 | 上报时限 |
| ---- | ---- | -------- |
| P1 | 大规模账号泄露、核心系统被入侵、敏感数据批量外泄 | 15 分钟内 |
| P2 | 单个账号异常、疑似钓鱼成功、少量敏感信息误发 | 30 分钟内 |
| P3 | 收到可疑邮件、发现异常链接、设备短暂离身 | 2 小时内 |

## 钓鱼邮件处理

收到疑似钓鱼邮件时，不要点击链接、不要下载附件、不要回复邮件。员工应在邮件客户端中点击“举报钓鱼”按钮，或将邮件作为附件转发给安全响应团队。已经点击链接或输入账号密码的，需要立即修改密码并联系安全响应团队。

## 账号异常处理

如果发现账号在异地登录、收到异常 MFA 验证、或系统提示存在风险登录，应立即执行以下操作：

1. 修改统一身份账号密码。
2. 检查 MFA 绑定设备是否被替换。
3. 退出所有已登录会话。
4. 在安全事件系统提交账号异常工单。
5. 等待安全响应团队确认是否需要冻结账号。

## 敏感数据误发

敏感数据误发包括把员工个人信息、客户资料、财务数据、代码密钥或内部经营数据发送给无权限人员。发现后应立即尝试撤回，并在 30 分钟内上报。不得自行要求收件人删除后就结束处理，必须由安全响应团队评估影响范围。

## 设备丢失

公司电脑、手机或含有公司数据的移动存储设备丢失后，应在 30 分钟内上报。员工需要提供设备编号、最后使用时间、可能丢失地点和设备是否开启磁盘加密。

## Agent 回答提示

当用户询问钓鱼邮件、异常登录、数据误发、设备丢失、安全上报时，优先引用本文。回答时需要说明上报时限、禁止操作和下一步处理动作。
